# Layer 2 SRO Checklist (v2)

Assess what kind of action the capability needs now:

- 🔧 `Stabilise` = restore control and reliability.
- 📐 `Rationalise` = reduce duplication and simplify.
- 🚀 `Optimise` = improve performance on a stable base.

Tick only observable signals.

Scoring guide per column: `0-4 weak`, `5-7 moderate`, `8+ strong`.

Use totals as directional signals, not proof. Observation quality outweighs score spread.

## 🔧 Stabilise

- Planned work is frequently interrupted.
- Priorities shift unpredictably.
- Backlog is dominated by urgent items.
- Recurring incidents or failures continue.
- Problems reappear without root-cause fix.
- Changes create fresh issues.
- Delivery depends on key individuals.
- Team stress or overload is persistent.
- Knowledge is mainly tribal.
- Standard process is absent or ignored.
- Metrics are missing or untrusted.
- Workarounds are common.

## 📐 Rationalise

- Multiple tools perform the same function.
- Duplicate workflows exist.
- Exceptions and special cases are widespread.
- Teams complete similar work differently.
- Standard process is unclear or fragmented.
- Outputs vary by person or team.
- Coordination overhead is high.
- Handoffs are frequent friction points.
- Dependencies create delays.
- Rework is common.
- Time is spent navigating complexity.
- Tool/vendor overlap is significant.

## 🚀 Optimise

- Work is predictable and rarely disrupted.
- Processes are standardised and followed.
- Systems are reliable and trusted.
- Metrics are reviewed routinely.
- Experiments are run deliberately.
- Continuous improvement rituals exist.
- Automation opportunities are identifiable.
- Cycle time is measured and improvable.
- Decisions are data-informed.
- There is capacity for improvement work.
- Improvement work is planned, not ad hoc.
- Performance gains are sustained over time.

## Decision rule

1. Identify the strongest recurring behaviour pattern first.
2. Count ticks by column as a secondary sense-check.
3. Highest score indicates dominant action need.
4. If top two columns are within 2 ticks, treat as mixed.
5. In mixed states, default sequence is:
   - 🔧 Stabilise → 📐 Rationalise → 🚀 Optimise

## Guardrails

- 🚨 Do not 🚀 optimise 🔥 instability.
- 🚨 Do not 📐 rationalise before shared understanding (if Layer 1 is 🟡 Align).
- 🚨 Do not scale learning until 🔍 Probe uncertainty is reduced.
