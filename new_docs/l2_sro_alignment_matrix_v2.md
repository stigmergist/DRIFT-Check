## What each behaviour type means

- 🔧 Stabilise: incident control, variability reduction, predictable execution.
- 📐 Rationalise: simplification, standardisation, consolidation.
- 🚀 Optimise: performance improvement, automation, experimentation.

## Default sequence

- 🔧 Stabilise → 📐 Rationalise → 🚀 Optimise

## Fast guardrails

- 🔥 If incidents are recurring, 🚀 optimise is usually premature.
- 🟡 If teams disagree on the problem, 📐 rationalisation is likely fragile.
- 🔍 If cause-effect is unclear, 🚀 optimise scales uncertainty.
- ⛔ If value is unclear, any optimisation is likely waste.
