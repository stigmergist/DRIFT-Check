# External Validity and Value Drift

## Why this document exists
Teams often improve systems that are internally coherent, efficient, and aligned — but no longer relevant to their environment.

This creates a clarity gap between:
- **internal performance signals** (things look good)
- **external reality** (value is weakening or shifting)

Decisions that improve with this understanding:
- whether to continue investing in improvement vs stop or pivot  
- whether performance issues are actually value issues  
- when to challenge the system itself, not just how well it runs  

---

## Core claim
A system can be internally effective while externally misaligned.  

If customer behaviour, demand, or usage does not move with internal improvement, the issue is likely **value drift**, not capability.  

At that point, continued optimisation increases waste rather than impact.

---

## What this is (and is not)
- Helps identify when **good execution is applied to the wrong thing**
- Helps distinguish **performance gaps vs relevance gaps**
- Helps decide when to shift from improving → questioning → stopping  

Not:
- A performance optimisation guide  
- A capability maturity model  
- A replacement for product/market validation  

Boundary:
- Capability explains **how well something works**  
- External validity explains **whether it should exist or continue**

---

## Key questions this document answers
1. How do we detect “internally good, externally wrong”?
2. Which external signals matter most?
3. When is this a value problem vs a capability problem?
4. When should we stop improving and reassess?
5. How does value drift show up before it becomes obvious?

---

## Observable signals

- Signal:
  - What to look for: Internal metrics improve (efficiency, quality, speed) but customer behaviour does not change
  - Why it matters: Improvement is decoupled from value creation
  - Common misread: “We just need to improve more”

- Signal:
  - What to look for: Increasing effort required to justify, sell, or explain the work
  - Why it matters: Value is no longer self-evident
  - Common misread: “Stakeholders don’t understand”

- Signal:
  - What to look for: Usage becomes concentrated in lower-value segments or unintended use cases
  - Why it matters: Core value proposition is weakening
  - Common misread: “Growth is still happening, so it’s fine”

---

## Decision implications
- If these signals are present, likely action is:
  - Shift from improving → questioning value (Stop-context emerging)

- If confidence is low, safest next step is:
  - Test whether outcomes are actually used or create impact

- If value is unclear, default is:
  - Reduce or pause investment rather than optimise further

- Risks of acting too early:
  - Abandoning something that still has recoverable value

- Risks of acting too late:
  - Scaling and embedding something that no longer matters

---

## Failure patterns

- Pattern:
  - Typical symptom: Continuous improvement programmes with little external impact
  - Underlying cause: Treating value problems as capability problems
  - Consequence if ignored: Efficient delivery of irrelevant outputs

- Pattern:
  - Typical symptom: Strong internal alignment but declining demand or engagement
  - Underlying cause: System optimised around outdated assumptions
  - Consequence if ignored: High-performing system with no meaningful outcome

---

## Practical tests
1. Fast test to validate interpretation  
   → If this worked perfectly, would customer behaviour meaningfully change?

2. Fast test to separate signal from noise  
   → Are improvements visible outside the system, or only inside it?

3. Fast test to choose between two actions  
   → Is the constraint performance — or relevance?

---

## Trade-offs and tensions

- Tension: Continue improving vs challenge the system itself
  - Over-correction risk: Stopping too early and losing viable capability
  - Under-correction risk: Investing heavily in something losing relevance
  - How to balance in practice: Tie improvement directly to observable external change

---

## Example scenarios

### Scenario A
- Context: A product team improves onboarding flow conversion rates
- Observable signals: Conversion improves, but long-term retention does not
- Interpretation: Internal optimisation not translating to real value
- Action chosen: Reassess value proposition rather than further optimise funnel
- What happened: Identified mismatch between product promise and actual need

### Scenario B
- Context: Operations team reduces cost and increases delivery speed
- Observable signals: Customer demand remains flat, pricing pressure increases
- Interpretation: System efficient but value declining
- Action chosen: Pause optimisation, reassess market relevance
- What happened: Shifted focus to different segment with stronger demand
