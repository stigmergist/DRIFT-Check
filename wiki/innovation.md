# Innovation

Innovation in DRIFT is treated as exploration under uncertain cause and effect.

Early innovation work is often discontinuous: success criteria are not yet proven, value assumptions are still hypotheses, and outcomes vary between attempts. In this phase, innovation should not be managed as if it were already performance work.

The main risk is premature convergence. Organisations optimise, standardise, or scale too soon because there is pressure for certainty. That can lock weak solutions in place and suppress learning.

Healthy innovation in DRIFT means bounded probing, explicit uncertainty, and clear transitions. Move from learning mode to performance mode only when reliability signals justify it.

Use the [innovation spiral](innovation_spiral.md) to structure this shift. Early loops should test and reject quickly. Later loops should deepen confidence and commit only after complete turns across solution-quality dimensions.
