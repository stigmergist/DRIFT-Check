# Companion Docs in Practice

The docs in [docs](../docs) are practical companions for manual DRIFT use.

The sequence is simple. First read context signals (Proceed, Align, Probe, Stop). Then identify dominant capability action need (Stabilise, Rationalise, Optimise). Then test fit before scaling action.

The checklists are directional tools, not certainty engines. Observable behaviour should outweigh raw scores.

Used well, these docs help teams have faster, clearer conversations. Tick counts help, but they are not judgement. Use these sheets in working sessions, board reviews, and pulse checks to focus discussion before large decisions.

This is also the philosophy behind [drift_check.md](drift_check.md) — using a lightweight tool to sharpen perception, not as a framework to apply rigidly.
