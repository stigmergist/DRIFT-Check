# Scaling

Scaling amplifies how a capability behaves; it does not fix how it behaves.

This is one of the most practical DRIFT warnings. If a capability is reliable, scaling can create more value. If it is unstable or misaligned, scaling distributes failure faster and wider.

Teams often experience pressure to scale as proof of success. DRIFT reframes scale as a force multiplier that should be applied only after fit checks are strong enough.

A useful question before scaling is: what pattern are we about to amplify? That question should be paired with [decision thresholds](decision_thresholds.md): what level of confidence is required before amplification is justified.
