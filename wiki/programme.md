# Programme

A programme is a hypothesis about how change will create value.

This framing is intentionally demanding. If a programme is a hypothesis, it must be testable in reality, not just governable in process. It should generate observable evidence that capabilities are improving and that outcomes are being used to create value.

DRIFT challenges a common failure mode: programmes that are operationally busy and governance-compliant but weakly connected to real value creation. In that pattern, reporting improves while outcomes remain inconsistent or unused.

Good programme judgement asks three linked questions repeatedly. Is the context still fit to act? Is the capability intervention still fit to state? Is value still being created in the environment that matters?

Programmes also need explicit controls for [parallelism](parallelism.md) and [investment vs fit](investment_vs_fit.md). Without them, programmes can become coordination-heavy while outcomes remain unchanged.

It also helps to keep planning language clear. A roadmap is a coordination artifact, not the change itself. The journey is what the system actually experiences. The road is the path currently available under real constraints.

Programmes always operate within constraints of [agency.md](agency.md) — not everything can be changed. Identifying where change is possible and where constraints must be navigated is part of good programme thinking.
