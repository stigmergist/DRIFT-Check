# Rationalise

Rationalise means reducing unnecessary complexity once basic reliability is present.

Complexity in DRIFT terms is when the same outcome is achieved in many different ways. Some variation is useful. But excessive variation increases handoff cost, coordination overhead, and rework.

Rationalisation removes duplication, aligns methods, and simplifies structures where simplification increases coherence. It should not be used as a substitute for **[stabilisation](stabilise.md)**. Simplifying unstable systems can remove buffers and increase **[fragility](fragility.md)**.

A practical test: if teams can produce similar outcomes through fundamentally different routes, rationalisation likely has value. If the same route produces unpredictable outcomes, stabilisation is usually needed first.
